# MaydayLand · 五月天·城市漫游 微信小程序 MVP 功能需求文档

> 报名赛道：**生活娱乐**　|　产品形态：**微信小程序**　|　创意名称：**五月天·城市漫游**

## 1. 项目概述

### 1.1 项目背景（创新性：需求创新）
MaydayLand 是面向五月天粉丝（五迷）群体的微信小程序，核心创意是 **"跟着歌词，发现城市里的五迷角落"**。

五月天的歌词陪伴了无数人的青春，许多城市里藏着五迷开的咖啡馆、书店、涂鸦墙、唱片行——它们散落各处，**长期缺乏一个集中的发现入口**。同时，五迷在演唱会期间和日常生活中存在 **极强的同好社交需求** 和 **对官方/同好信息的渴望**，但现有平台（微博、大众点评、票务平台）无法提供"音乐主题 + 城市漫游 + 同好认证"的统一入口。

本项目基于微信小程序生态，打造一款轻量级、强传播、有情绪的 MVP 产品，将 **"歌词主题打卡"** 与 **"演唱会同好互动"** 融为一体，验证五迷群体对 **"歌词角落地图 + 打卡分享 + 同好暗号"** 三大核心功能的接受度。

### 1.2 项目目标
1. **验证创意假设**：五迷愿意为"歌词主题探店打卡"产生分享传播（分享率 ≥ 30%）。
2. **实现用户增长**：通过分享裂变 + 演唱会节点投放，上线 1 个月内用户数达 30,000+，覆盖 5 个核心城市（台北、上海、北京、广州、成都）。
3. **建立功能闭环**：实现 "**发现歌词角落 → 到店打卡 → 生成卡片分享 → 同好暗号互动**" 的完整用户旅程。
4. **沉淀 UGC 内容**：上线 1 个月内由用户共创推荐"歌词角落" ≥ 500 个。

### 1.3 项目范围

**包含功能（MVP）**：
- 🗺️ **歌词地图**：基于 LBS 的"歌词角落"探店地图与列表
- 📸 **打卡卡片**：到店打卡生成"歌词 + 照片"专属分享卡片
- 💬 **五迷暗号**：店铺暗号兑换 + 同好身份识别
- 🎤 **演唱会信息聚合**：日程、场馆地图、官方动态
- 🔄 **微信分享裂变**：携带用户 ID + 卡片图的社交分享

**排除功能（非 MVP）**：
- 复杂的用户注册登录系统（采用微信原生授权 + 本地缓存）
- 实时弹幕同步、IM 聊天
- 会员等级、积分商城、支付交易
- 商家入驻后台、订单系统
- 第三方数据埋点平台对接

## 2. 用户分析

### 2.1 目标用户画像
| 维度 | 特征描述 |
| :--- | :--- |
| **核心用户群体** | 18-35 岁五月天粉丝，女性占比约 65%，男性 35% |
| **生活方式** | 喜欢 citywalk、独立咖啡馆、独立书店、复古唱片店 |
| **消费能力** | 单场演唱会平均消费 3,000 元+，月度探店预算 500-1,500 元 |
| **核心需求** | 一站式发现"歌词主题角落" + 同好认证 + 仪式感分享 |
| **高频场景** | 周末 citywalk、出差/旅行、演唱会前后续场、纪念日打卡 |

### 2.2 用户需求与痛点

#### 2.2.1 核心需求
1. **歌词角落发现需求**：在陌生或熟悉的城市里，找到与五月天歌词气质契合的店铺、空间、街角。
2. **同好认证需求**：希望快速识别"这家店是真五迷据点"，避免踩雷。
3. **仪式感分享需求**：日常打卡照片千篇一律，五迷渴望"歌词 + 我"的专属仪式感。
4. **演唱会延伸需求**：演唱会前/后想找附近的五迷据点"续场"、找同好同行。
5. **共创表达需求**：希望推荐自己发现的"歌词角落"给更多五迷。

#### 2.2.2 主要痛点（对应创新性维度「需求创新」）
1. **信息散落**：歌词主题店、五迷据点、演唱会信息分散在微博、小红书、大众点评等渠道，**没有一张专属于五迷的城市地图**。
2. **缺少"同好滤镜"**：大众平台无法筛出"真·五迷据点"，粉丝担心踩雷、氛围不对。
3. **打卡分享缺乏仪式感**：现有打卡工具不能体现"五月天 × 我 × 这座城市"的情感连接，照片千篇一律。
4. **孤独感**：一个人 citywalk 无聊，缺少同好组队/约伴入口。
5. **演唱会信息割裂**：场馆、接驳车、周边美食/书店没有统一地图，续场体验 fragmented。

#### 2.2.3 为什么现有产品无法满足（对应创新性维度「解决思路」）

| 现有产品 | 能提供的 | 无法提供的 |
|----------|----------|------------|
| 微博/小红书 | UGC 推荐、图片种草 | 结构化地图、同好认证、暗号权益 |
| 大众点评 | 商户信息、评分 | 歌词主题匹配、五迷氛围筛选 |
| 票务平台 | 演唱会日程 | 场馆周边五迷角落、续场路线 |
| 普通打卡 App | 地点签到 | 歌词情绪标签、专属卡片模板、粉丝身份徽章 |

MaydayLand 通过 **"歌词情绪标签 → 城市角落 → 同好暗号 → 仪式感分享"** 的闭环，把分散的粉丝行为整合到一个产品内，这是现有产品没有做过的。

## 3. 功能需求

### 3.1 核心功能一：歌词地图（🗺️ 发现歌词角落）

#### 3.1.1 功能描述
基于 LBS 展示用户所在城市的"五迷角落"（咖啡馆、书店、涂鸦墙、唱片店、演唱会场馆等），每个角落配一句契合气质的五月天歌词。

#### 3.1.2 核心功能点
- **地图展示**：调用 `map` 组件 + 自定义气泡 markers，展示附近"歌词角落"。
- **标签筛选**：支持筛选 `咖啡馆 / 书店 / 唱片店 / 涂鸦墙 / 演唱会场馆 / 新开的店`。
- **歌词关键词搜索**：输入"温柔/倔强/知足/憨人"等情绪关键词，匹配气质契合的角落（MVP 阶段使用预置标签匹配，非真实 AI）。
- **角落详情**：地点、配图、匹配歌词、五迷推荐语、暗号、UGC 笔记数。
- **新角落共创**：用户可上传自己发现的角落（图片 + 地址 + 推荐理由 + 推荐歌词），由后台审核后上架。

#### 3.1.3 数据来源
- **种子数据**：项目方初始整理 5 个核心城市各 50 个"歌词角落"，托管在 CDN（`corners.json`）。
- **用户共创**：用户提交的角落通过云开发数据库 `user_corners` 暂存，运营审核后并入主数据。

---

### 3.2 核心功能二：打卡卡片（📸 仪式感分享）

#### 3.2.1 功能描述
用户到达"歌词角落"后，可上传一张现场照片，自动生成 **"歌词 + 我的照片 + 地点 + 日期"** 的专属打卡卡片，可一键保存或分享。

#### 3.2.2 核心功能点
- **到店核验**：用 `wx.getLocation` 校验用户与角落距离 ≤ 200 米才允许打卡（MVP 可降级为"宽松校验 + 1km"）。
- **照片上传**：调用 `wx.chooseMedia` 选择/拍摄一张照片，本地处理。
- **卡片合成**：使用 `canvas 2d` 在前端合成卡片图（背景渐变 + 用户照片 + 歌词字幕 + 日期 + 地点徽章 + 小程序码）。
- **卡片样式**：提供 3 套主题模板（复古拍立得 / 演唱会票根 / 极简胶片），契合五月天专辑视觉。
- **保存与分享**：`wx.canvasToTempFilePath` 导出卡片，支持保存到相册或分享给微信好友/朋友圈。
- **打卡足迹**：用户主页展示打卡过的角落清单（本地存储 + 云端兜底）。

---

### 3.3 核心功能三：五迷暗号（💬 同好认证）

#### 3.3.1 功能描述
每个"歌词角落"绑定一个 **专属暗号歌曲名**（如"倔强""温柔""憨人"），用户在店内对暗号可获得隐藏权益（折扣、限定饮品、明信片等）。同时暗号也是同好之间的"接头方式"。

#### 3.3.2 核心功能点
- **暗号展示**：进入角落详情页可见暗号提示（如 `📌 五迷暗号：倔强`）。
- **暗号兑换**：用户在店内点击"我要对暗号"按钮 → 显示用户专属二维码（含一次性 token） → 店家小程序扫码核销。
- **核销记录**：核销成功后用户获得"五迷徽章"，沉淀在个人主页。
- **MVP 简化方案**：商家端使用现成的 `wx.scanCode` + 云函数校验 token；初期 5-10 家合作店铺。
- **同好识别**：用户主页展示徽章墙（已对暗号的角落数 / 城市数 / 主题歌覆盖数）。

### 3.4 核心功能四：演唱会信息聚合（🎤 主场景延伸）

保留并强化原 MVP 中的演唱会信息能力，让"日常城市漫游"在演唱会节点形成流量爆发。

#### 3.4.1 核心功能点
- **日程展示**：轮播图展示近期演唱会城市/场馆/日期。
- **官方动态**：列表展示天气提醒、接驳车、安检通知等。
- **场馆地图**：基于 `map` 组件标注场馆位置 + 周边 1 公里内的"歌词角落"，引导"演唱会前 citywalk"。
- **现场互动**：演唱会期间页面顶部出现"今晚同场"标签，进入查看现场五迷的留言（仍采用本地存储 + 分享同步方案，避免后端复杂度）。

---

### 3.5 核心功能五：评论互动 & 微信分享裂变

#### 3.5.1 评论互动（轻量化）
- 角落详情、演唱会页面均支持发布短评（≤ 200 字）。
- 评论本地存储（`wx.setStorage`，单用户最多 10 条/页面），通过分享链接同步给好友。
- 进入页面时合并 CDN 公共评论 + 本地评论 + 链接携带评论。

#### 3.5.2 评论点赞与回复（v1.1 增强）
- **点赞**：每条评论右侧心形按钮，点击切换点赞状态，本地记录 `likedCommentIds`，显示累计点赞数。
- **回复评论**：支持二级回复（评论 → 回复），点击评论项弹出回复输入框，回复以缩进形式挂在父评论下。
- **数据结构**：
  ```json
  {
    "id": "cmt_001",
    "cornerId": "corner_001",
    "user": "倔强的番茄",
    "text": "拿铁拉花超美！",
    "time": "2026-06-18 14:32",
    "likes": 12,
    "replies": [
      { "id": "rpl_001", "user": "明园老板", "text": "谢谢支持～", "time": "2026-06-18 15:10", "likes": 3 }
    ]
  }
  ```
- **存储上限**：单角落评论本地保留 50 条，超出滚动归档；点赞数与回复数合并展示。

#### 3.5.3 微信分享裂变
- 分享主路径：**打卡卡片图** + 文案 `"我在 {城市}{角落}，遇见了一句 {歌词}。MaydayLand 等你来"`。
- 分享链接携带 `userId` + `cornerId`，新用户进入直达对应角落详情页。
- 朋友圈分享自动调用卡片图作为封面。
- 分享统计依赖微信小程序原生「分享分析」面板。

### 3.6 核心功能六：首页城市切换（🏙️ 多城漫游）

#### 3.6.1 功能描述
首页顶部增加城市切换入口，用户可在 5 个核心城市（台北 / 上海 / 北京 / 广州 / 成都）之间切换，列表与地图仅展示当前选中城市的角落与演唱会，避免信息过载。

#### 3.6.2 核心功能点
- **城市选择器**：首页顶部 Banner 下方增加 `🏙️ 当前城市：北京 ▾` 入口，点击弹出城市选择面板（半屏抽屉）。
- **记忆选择**：用户切换城市后写入 `localStorage.currentCity`，下次进入自动恢复。
- **数据联动**：
  - 角落列表 / 地图标记 / 演唱会日程均按当前城市过滤。
  - 当城市无种子角落时，显示空状态 + "去推荐一个角落" CTA。
- **定位兜底**：首次进入调用 `wx.getLocation` 反查城市，失败则默认"北京"。
- **城市徽章**：每个城市配专属色调徽章（台北粉 / 上海蓝 / 北京黄 / 广州绿 / 成都紫）。

#### 3.6.3 交互细节
- 城市选择面板支持搜索框（输入城市名快速定位）。
- 切换城市时列表淡入动画（200ms），避免突兀。
- 顶部状态栏同步显示当前城市名 + 城市色调点缀。

---

### 3.7 核心功能七：角落详情增强（🗺️ 地图跳转 + 导航）

#### 3.7.1 功能描述
角落详情页增加更丰富的描述信息，并提供"查看大地图"与"一键导航"能力，让用户从发现角落到实地到店的链路无缝衔接。

#### 3.7.2 核心功能点
- **详情增强字段**：
  - `description`：角落故事长描述（200-400 字），讲述店铺与五月天的渊源。
  - `openingHours`：营业时间（如 `10:00-22:00`）。
  - `phone`：联系电话（可点击拨打）。
  - `tags`：氛围标签（如 `#复古 #黑胶 #咖啡香`）。
  - `tips`：到店小贴士（如 `对暗号可获隐藏菜单`）。
- **地图跳转**：详情页增加"查看大地图"按钮，点击打开全屏地图页（独立 page），展示角落位置 + 周边同类角落。
- **一键导航**：详情页底部固定按钮组增加"🧭 导航"按钮：
  - 小程序端：调用 `wx.openLocation` 唤起微信内置地图导航。
  - HTML Demo：调用 `window.open('https://maps.apple.com/?ll=...')` 或高德/百度地图 URL Scheme。
- **地址复制**：长按地址可复制到剪贴板，方便分享给好友。

#### 3.7.3 交互细节
- 详情页顶部图片下方增加信息卡（营业时间 / 电话 / 距离）。
- "导航"按钮与"我要打卡""我要对暗号"并列，形成三按钮组。
- 大地图页支持点击标记跳回对应角落详情。

---

### 3.8 核心功能八：歌单解锁（🎵 五月天歌单收集）

#### 3.8.1 功能描述
将"打卡 / 对暗号 / 分享 / 城市切换"等用户行为转化为"解锁五月天歌曲"的进度，构建一个轻量级的"歌单收集"游戏化机制，提升用户粘性与探索动力。

#### 3.8.2 核心功能点
- **歌单墙**：我的页新增"歌单解锁"区块，展示五月天 9 张专辑的代表歌曲（共 24 首精选），未解锁灰锁、已解锁高亮。
- **解锁条件**（多维度触发）：
  | 解锁动作 | 解锁歌曲示例 |
  | :--- | :--- |
  | 首次打卡任意角落 | 《任意门》 |
  | 对暗号成功 1 次 | 对应暗号歌曲（如《倔强》） |
  | 分享卡片 1 次 | 《知足》 |
  | 切换城市 3 次 | 《离开地球表面》 |
  | 累计打卡 5 个角落 | 《干杯》 |
  | 累计打卡 5 个城市 | 《人生海海》 |
  | 评论被点赞 10 次 | 《温柔》 |
  | 完成 1 次附近打卡（1km 宽松圈） | 《突然好想你》 |

## 4. 技术实现方案

### 4.1 交付形态与前端框架

**最终交付形态：交互式可体验的 HTML 格式文件（Zip 打包上传社区）**

为满足比赛"交互式可体验 HTML"的交付要求，本 MVP 采用 **双形态** 策略：

| 形态 | 用途 | 技术栈 |
| :--- | :--- | :--- |
| **HTML 交互 Demo（本次交付）** | 社区上传 / 浏览器即开即体验 / 评审可点击 | 原生 HTML5 + CSS3 + Vanilla JS + Canvas 2D，单文件自包含 |
| 微信小程序（正式上线形态） | 真实分发 / 微信生态裂变 | WXML / WXSS / JS + canvas 2d |

**HTML Demo 设计原则**：
- **手机外壳模拟**：以 iPhone 比例（375×812）渲染小程序框架，底部 TabBar 还原微信小程序交互。
- **零依赖自包含**：所有 CSS/JS/数据内联，图片走 CDN（Unsplash），双击 `index.html` 即可体验。
- **功能完整可点击**：五大核心功能（歌词地图 / 打卡卡片 / 五迷暗号 / 演唱会聚合 / 分享裂变）均可在浏览器中真实操作。
- **数据真实新鲜**：复用 2026 年最新五迷打卡点 / 演唱会日程，无事实错误。
- **视觉风格统一**：服务五月天五球、卜卜、STAYREAL 的潮玩治愈风（蓝/粉/黄/绿/紫五色 + 圆润几何）。

**正式上线形态（微信小程序原生）**：MVP 重视性能与上线速度，避免引入 Taro / uni-app 带来的体积与兼容成本。

## 6. 上线与迭代策略

### 6.1 上线准备
- 配置小程序合法域名，添加 CDN 静态资源域名 + 云开发资源。
- 5 个核心城市 × 50 个种子角落数据采集与人工审核。
- 5-10 家首批合作店铺洽谈，配置专属暗号与权益。
- 设计 3 套打卡卡片模板（拍立得 / 票根 / 胶片）。
- 准备分享文案、首页 banner、加载占位图。

### 6.2 迭代路线
1. **第一阶段（1-2 周）**：上线"歌词地图 + 打卡卡片 + 演唱会信息"主线，验证基础留存与分享率。
2. **第二阶段（3-4 周）**：开放"五迷暗号"商家核销，扩大种子城市到 8-10 个，开放用户共创角落。
3. **第三阶段（5-8 周）**：根据数据决定是否扩展"同城约伴 citywalk""演唱会现场弹幕同步"等高复杂度功能。

**数据驱动决策**：
- 若分享率 ≥ 30% 且分享新增用户占比 ≥ 50% → 验证社交传播假设成立。
- 若角落打卡率 ≥ 25% 且评论发送率 ≥ 3 条 / 用户 → 验证创意主线假设成立。
- 若 UGC 共创 ≥ 500 个 → 验证用户共创动力，可启动"城市合伙人"运营计划。
- 若 DAU/MAU 未达目标 → 优化首页歌词推荐算法、首屏热门角落，强化"演唱会节点"投放。

## 7. 风险控制

### 7.1 技术风险
- **本地存储容量限制（最大 2MB）**：评论 / 打卡足迹采用「本地最多 10 条 + 云端兜底」策略，超出自动归档。
- **Canvas 卡片合成兼容性**：iOS / Android 字体渲染差异，使用系统字体 + 充分真机测试。
- **LBS 定位偏移**：到店核验范围放宽至 200m-1km，并允许"无定位也可生成纪念卡片（不计入打卡足迹）"。

### 7.2 业务风险
- **角落数据失实**：用户共创角落需运营审核 + "信息有误"反馈通道；店铺关闭/迁址有用户举报机制。
- **演唱会信息更新不及时**：CDN JSON 配合企业微信群机器人推送更新，运营可在 5 分钟内推送热修复。
- **暗号被刷/滥用**：一次性 token + 云函数校验时效（10 分钟内有效）+ 单用户日限 1 次。

### 7.3 版权与合规风险
- **歌词使用合规**：仅展示歌词片段（≤ 30 字），不提供完整歌词；标注词曲作者；优先使用公开报道中已大量传播的金句。
- **店铺名称使用**：与合作店铺签订合作协议；非合作店铺仅作为用户共创推荐展示，需店家可申请下架。
- **用户照片合规**：上传时明确告知"仅用于生成本人打卡卡片"，不在公开列表展示用户原图。

### 7.4 运营风险
- **冷启动城市覆盖不足**：首期聚焦 5 城（台北 / 上海 / 北京 / 广州 / 成都），不盲目铺开；演唱会城市优先扩展。
- **分享传播不及预期**：A/B 测试不同分享文案、卡片模板；演唱会节点配合 KOL 五迷投放。

## 8. 总结

本版 MaydayLand 微信小程序 MVP 围绕 **"五月天·城市漫游"** 创意主线，遵循 **"最小化、可行化、产品化"** 原则，将 **歌词地图、打卡卡片、五迷暗号** 三大创新功能与 **演唱会信息聚合、轻量评论、微信分享裂变** 基础能力结合，构建起"发现 → 打卡 → 分享 → 认证"的完整闭环。
