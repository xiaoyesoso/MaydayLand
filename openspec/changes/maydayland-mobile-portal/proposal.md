# MaydayLand 移动端统一门户 · 变更提案

## 背景

MaydayLand（justloveit.cn）当前为 PC 端导航站形态，11 个子产品分散部署在不同子域名/非标准端口：

- 8 款游戏：猜词、猜歌名、二选一、Emoji 猜歌、十秒猜歌、歌词选择、合成大月天、记忆配对
- 3 款工具：朋友圈、DIY 刮刮乐、图库

移动端适配不足、无统一用户体系、各游戏排行榜互不关联、`:4433` 端口在移动网络下易被拦截。

## 目标

在 **当前 Flask H5 单页应用** 技术栈内（原生 HTML + ES5 JS + CSS，零依赖），新增一个可独立访问的移动端门户入口，整合 11 款子产品，提供：

1. 统一首页导航（游戏网格 + 工具入口）
2. 统一用户昵称与本地战绩（localStorage）
3. 每款游戏/工具至少可玩的 MVP 页面
4. 复用现有品牌素材（`assets/MaydayLand/images/`）
5. 移动端优先布局（最大宽度 480px 居中）

## 范围

**In Scope**
- 新建门户首页 `/mobile-portal`
- 8 款游戏的 H5 单文件实现（首版可本地 playable，不强制后端）
- 3 款工具的 MVP 页面（朋友圈信息流、DIY 刮刮乐、图库瀑布流）
- 本地排行榜与战绩（localStorage）
- 复用现有素材

**Out of Scope（本变更不做）**
- 后端统一用户体系 / 微信登录
- 服务端排行榜持久化
- 原生 App / 小程序原生工程
- PWA Service Worker（后续阶段）
- 朋友圈 UGC 发布后端

## 成功标准

1. 移动端门户首页能完整展示 11 个入口
2. 每款游戏至少完成核心玩法一局可玩
3. 在 iOS Safari / 微信内置浏览器 / Android Chrome 正常渲染
4. 总包体积 ≤ 500KB（不含图片）
5. 首屏加载时间 ≤ 2s（本地 4G 模拟）
